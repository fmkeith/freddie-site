First day. I wasn't sure what to expect — I thought work experience at a web dev studio would mainly be watching someone else type really fast and nodding along. It wasn't like that at all.

Uncle Stu started by asking me what I thought a website actually was. I said something about pages on the internet. He said yes, but what *is* it — like, if you stripped everything back? We spent about half an hour on that. A website is just files on a computer somewhere, and when you type an address into a browser, the browser fetches those files and turns them into something you can see. That was actually kind of mind-blowing to hear explained plainly.

We installed VS Code and set up my laptop. The terminal was the weird bit — I've never used one before and it just looks like hacking in a film, but Stu explained it's just a way of talking to the computer in text instead of clicking things. We ran `node --version` and `npm --version` to check they were installed. They were. Small win.

Then we looked at a basic HTML file — just a `.html` file opened in a browser. Stu wrote a `<h1>` and a `<p>` and showed me how saving the file and refreshing the browser makes the change appear. I tried changing the text and it worked instantly. That felt good.

By the end of the day I had VS Code set up, a GitHub account sorted, and a rough idea of the plan: spend three days building a real website that would actually go live on the internet. That last bit still felt abstract.
